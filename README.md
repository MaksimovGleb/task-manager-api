# Task Manager API (Laravel)

REST API для управления списком задач.

## Функционал
- Создание, редактирование, удаление и просмотр задач.
- Поиск по названию.
- Сортировка по дате выполнения или дате создания.
- Постраничный вывод (Pagination).
- Документация API через Swagger.

## Технологии
- **PHP 8.2**
- **Laravel 12**
- **SQLite** (по умолчанию)
- **Docker**
- **Swagger (L5-Swagger)**

## Установка и запуск (Локально)

1. Установите зависимости:
   ```bash
   composer install
   ```
2. Настройте окружение:
   ```bash
   cp .env.example .env
   php artisan key:generate
   ```
3. Запустите миграции:
   ```bash
   php artisan migrate
   ```
4. Запустите сервер:
   ```bash
   php artisan serve
   ```

## Запуск в Docker

Для запуска в контейнерах (используя Laravel Sail):
```bash
./vendor/bin/sail up -d
./vendor/bin/sail artisan migrate
```

## API Методы

### Задачи (`/api/tasks`)
- `POST /api/tasks` — Создание задачи.
- `GET /api/tasks` — Список задач (параметры: `search`, `sort`, `page`).
- `GET /api/tasks/{id}` — Получение конкретной задачи.
- `PUT /api/tasks/{id}` — Обновление задачи.
- `DELETE /api/tasks/{id}` — Удаление задачи.

### Документация
Swagger доступен по адресу: `http://localhost:8000/api/documentation`

## Тестирование
Для запуска тестов используйте:
```bash
php artisan test
```

### Как я тестировал результат
1. **Автоматизированное тестирование**:
   - **Unit-тесты**: проверка бизнес-логики моделей.
   - **Feature-тесты**: тестирование контроллеров и корректности ответов API (статус-коды 200, 201, 422, 404). Проверка валидации обязательных полей.
2. **Инструменты**:
   - **PHPUnit**: стандартная библиотека Laravel для тестов.
   - **Swagger UI**: для ручного тестирования запросов в браузере.
   - **Postman/Insomnia**: для отладки сложных цепочек запросов.
3. **Методология**:
   - Проверка поиска: создание нескольких задач и фильтрация по подстроке.
   - Проверка пагинации: создание 15+ задач и проверка переключения страниц.
   - Валидация: отправка пустых заголовков или некорректных дат.
