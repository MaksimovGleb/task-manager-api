# Task Manager API (Laravel)

REST API для управления списком задач.

## Функционал
- Создание, редактирование, удаление и просмотр задач.
- Поиск по названию.
- Сортировка по дате выполнения или дате создания.
- Постраничный вывод (Pagination).
- Документация API через Swagger.

## Технологии
- **PHP 8.2**
- **Laravel 12**
- **SQLite** (по умолчанию)
- **Docker**
- **Swagger (L5-Swagger)**
- **PHPUnit** (для тестов)

## Установка и запуск (Docker)

Запуск приложения, выполнение миграций, сидирование базы данных и прогон тестов происходят автоматически при старте контейнера.

1. Соберите и запустите контейнеры:
   ```bash
   docker-compose up -d --build
   ```

API будет доступно по адресу: `http://localhost:8000/api`

Для повторного запуска тестов вручную:
```bash
docker-compose exec app php artisan test
```

## API Методы

### Задачи (`/api/tasks`)
- `POST /api/tasks` — Создание задачи.
- `GET /api/tasks` — Список задач (параметры: `search`, `sort`, `page`).
- `GET /api/tasks/{id}` — Получение конкретной задачи.
- `PUT /api/tasks/{id}` — Обновление задачи.
- `DELETE /api/tasks/{id}` — Удаление задачи.

### Документация
Swagger доступен по адресу: `http://localhost:8000/api/documentation`

## Тестирование
Для запуска тестов используйте:
```bash
php artisan test
```

### Как я тестировал результат
1. **Автоматизированное тестирование**:
   - **Feature-тесты**: тестирование контроллеров и корректности ответов API (статус-коды 200, 201, 422, 404). Проверка валидации обязательных полей, поиска, сортировки и пагинации.
2. **Инструменты**:
   - **PHPUnit**: стандартная библиотека Laravel для тестов.
   - **Postman**: для ручного тестирования запросов.
